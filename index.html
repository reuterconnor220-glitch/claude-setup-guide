Setup Guide

The complete walkthrough for configuring a new machine with Claude Code, full autonomy, plugins, agents, voice, and the Caruso Ventures deployment stack.

Begin Setup
1
Terminal

Download Warp

First things first — download Warp, a modern terminal built for developers. It has AI built in, command completion, and a much better UI than the default Mac terminal.

Download Warp for Mac →

Open the .dmg file, drag Warp to your Applications folder, and launch it. You'll use Warp for every step from here on out.

2
Foundation

Install Homebrew

Homebrew is the package manager for macOS. It's how we'll install everything else. Open Warp and paste this:

bash 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Heads up: On a fresh Mac, a popup will appear asking you to install "Command Line Developer Tools." Click Install and wait — this can take 10-20 minutes. It's normal. If it doesn't appear automatically, run xcode-select --install first.

After the install finishes, Homebrew will print some "Next steps" in the terminal. Run these two commands it tells you (they look like this):

bash 
echo >> ~/.zprofile
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

Verify it worked:

bash 
brew help

If you see a help page, you're good. If you see "command not found", close Warp completely, reopen it, and try again.

3
Runtime

Install Node.js

Node.js powers Claude Code and most modern web tooling. Install it via Homebrew:

bash 
brew install node

Verify it worked:

bash 
node -v

You should see a version number like v22.x.x. If you do, you're good.

4
The AI

Install Claude Code

Claude Code is Anthropic's CLI that lets Claude read your files, write code, run commands, and deploy — all from your terminal.

Primary Install (npm)

bash 
npm install -g @anthropic-ai/claude-code
If you see a "permission denied" or EACCES error: Try the backup install below instead. This is common on fresh Macs.

Backup Install (if npm gives you trouble)

If the command above fails, you can run Claude Code directly without a global install:

bash 
npx @anthropic-ai/claude-code

Authenticate

Now launch it for the first time:

bash 
claude

This will open a browser window where you sign in to your Anthropic account. Log in with your CV email. Once authenticated, Claude Code is linked to your account. Close the browser and return to the terminal.

Claude Code sees every file in the directory where you launch it. Navigate to a project folder before running claude to give it context.
5
Full Autonomy

Enable Dangerous Mode

By default, Claude Code asks for permission before running every command. Dangerous mode removes all permission gates so Claude can read, write, execute, and deploy without interruption.

First, create the AI workspace folder and add the aliases to your shell. Copy and paste this whole block — it does everything in one shot:

bash — paste this whole thing 
# Create the AI workspace
mkdir -p ~/AI

# Add the aliases to your shell config
echo 'alias claude="claude --dangerously-skip-permissions"' >> ~/.zshrc
echo 'alias claude-ai="cd ~/AI && claude --dangerously-skip-permissions"' >> ~/.zshrc

# Reload so the aliases take effect
source ~/.zshrc

Now skip the confirmation prompt in Claude's settings. This writes the complete settings file in one go:

bash — paste this whole thing 
mkdir -p ~/.claude
cat > ~/.claude/settings.json << 'EOF'
{
  "enabledPlugins": {
    "pr-review-toolkit@claude-plugins-official": true,
    "frontend-design@claude-plugins-official": true
  },
  "voiceEnabled": true,
  "skipDangerousModePermissionPrompt": true
}
EOF
This settings file also enables plugins and voice mode (covered in the next steps). We write it all at once so you don't have to edit it multiple times.

Starting a Session

  • claude — starts Claude in the current directory with full autonomy
  • claude-ai — jumps to your ~/AI workspace and starts Claude there
Dangerous mode means Claude can delete files, run any command, and modify anything without asking. Only use this on machines you trust.
6
Workflow

Bypass Mode vs. Plan Mode

Claude Code has two operating modes that control how much it does before checking in with you.

Bypass Mode

The default with dangerous mode. Claude reads files, writes code, runs commands, and deploys — all without pausing.

Best for fast builds

Plan Mode

Claude outlines its full approach before executing. You review, approve or adjust, then let it run.

Best for complex projects

How to Switch

  • Type /plan to enter plan mode
  • Type /exitplan to return to bypass mode

When to Use Which

  • Bypass: Straightforward tasks — "build me a landing page about X", "fix this bug", "add dark mode"
  • Plan: Multi-step projects with databases, auth, deployment, or anything where you want to review the architecture first
7
Superpowers

Enable Plugins

Plugins extend what Claude Code can do. Two are essential:

1. Frontend Design

Makes Claude generate beautiful, distinctive UIs instead of generic-looking sites. It thinks about typography, color palettes, animations, and spatial composition before writing code. Use it by typing /frontend-design followed by your request.

2. PR Review Toolkit

Gives Claude code review superpowers — it can analyze code quality, check test coverage, hunt for silent failures, review type design, validate comments, and simplify code. Use /review-pr to run a full review, or Claude will automatically use these agents when relevant.

How to Install

If you followed Step 5, these two are already enabled — we included them in the settings.json file we wrote. But here's how to browse and add more plugins in the future:

inside a claude session 
/plugins

This opens the plugin browser where you can see everything available and install with one click.

Not sure which plugins to grab? Ask Connor — he's tested most of them and can point you to the ones worth enabling for your workflow.

Army of Specialists

Install 156 Custom Agents

Agents are specialized personas that Claude can use for specific tasks — a software architect, a UX researcher, a sales coach, a security auditor, and 150+ more. We install them from an open-source collection by @msitarzewski on GitHub.

Step 1: Clone the Agent Repository

bash 
git clone https://github.com/msitarzewski/agency-agents.git ~/agency-agents-temp

Step 2: Copy Agents to Claude's Global Directory

This makes agents available in every project, not just one:

bash 
# Create the agents directory if it doesn't exist
mkdir -p ~/.claude/agents

# Copy all agent .md files into it
cp ~/agency-agents-temp/agents/*.md ~/.claude/agents/

# Clean up the temp clone
rm -rf ~/agency-agents-temp

Step 3: Verify

bash 
ls ~/.claude/agents/ | wc -l

You should see ~156 files. Claude now has access to all of these specialist agents and will automatically use the right one based on what you ask it to do.

Hands-Free

Enable Voice Mode

Voice mode lets you talk to Claude instead of typing. Describe what you want built, dictate changes, brainstorm while pacing around the room. If you followed Step 5, voice is already enabled in your settings.

How to Use Voice

  • Press the spacebar to start talking. Press it again to stop and send your message.
  • If the spacebar ever glitches or doesn't respond, type /voice to toggle voice mode on/off. This always works as a fallback.
Voice is great for iterating on something you can already see. Once Claude builds a page, just talk: "make the hero bigger", "swap the colors to dark mode", "add a section about pricing". Just talk naturally.
10
Version Control

Connect GitHub

GitHub stores your code and connects to Railway for automatic deployments.

Do this in a fresh Warp tab, NOT inside a Claude session. Open a new tab in Warp (Cmd+T) and run these commands directly in the terminal. The login flow works much better outside of Claude.

Install GitHub CLI & Git

bash 
brew install gh git

Set Up Your Identity

Git needs to know who you are. Replace with your name and CV email:

bash 
git config --global user.name "Your Name"
git config --global user.email "you@carusoventures.com"
git config --global init.defaultBranch main

Log In

This opens your browser to authenticate:

bash 
gh auth login

Log in with your CV email. Follow the browser prompts, then return to the terminal.

You generally don't need to create repos or push manually — just tell Claude "commit and push this to GitHub" and it handles everything in dangerous mode.
11
Deployment

Set Up Railway

Railway is where we deploy everything. It auto-detects your project type and deploys with one command.

bash 
brew install railway

Log in through the CLI. Try the browser login first:

bash 
railway login
The browser login often doesn't work. If it hangs or fails, use the browserless method instead — it gives you a code to paste into your terminal:
bash — use this if browser login fails 
railway login --browserless

Log in with your CV email.

Deploy a Project

bash 
# Navigate to your project
cd ~/my-project

# Create a new Railway project
railway init

# Deploy
railway up

For Static HTML Sites

Add a Dockerfile so Railway knows how to serve them:

Dockerfile 
FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Database

Set Up Supabase

Supabase is our PostgreSQL database with auth, storage, and real-time built in.

bash 
brew install supabase/tap/supabase

Log in through the CLI:

bash 
supabase login

Log in with your CV email. Then:

  1. Go to supabase.com/dashboard and create a new project
  2. Copy the project reference ID from the URL
  3. Link it locally:
bash 
supabase link --project-ref YOUR_PROJECT_ID

Connect in Your App

.env.local 
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
The default stack for all CV projects: Supabase (database + auth) + Railway (hosting) + GitHub (code).

Mastery

How to Build Great Things with Claude

Now that everything is set up, here's the playbook for getting the best results out of Claude Code. These are the patterns and prompts we've used to build everything at CV.

The Design → Build → Review Workflow

  1. Use /frontend-design when asking Claude to build anything visual. This activates the design plugin and produces dramatically better-looking output.
  2. Let Claude build it in bypass mode — just describe what you want and let it go.
  3. Run /review-pr after it's built to have Claude's review agents check the code. If the review passes, you're good. If it finds issues, tell Claude to fix them.

Start with a One-Sentence Brief

Be specific about what you want. The more context, the better the output:

"Build me a single-page explainer for our retirement planning app. Use a warm, natural color palette — forest greens and cream. Serif headings, sans-serif body. Include sections for what it does, how it works, and why it matters."

"Create a strategic briefing site for Boulder Startup Week 2026. Dark theme with navy and gold. Use glass-morphism effects. Include mermaid charts for the timeline and flowcharts for the event strategy."

Tell Claude How It Should Look

Specify the vibe — color palette, font pairing, and mood:

Navy + Gold Professional, executive, strategy-deck feel. Playfair Display headings + Inter body.
Forest + Cream Warm, approachable, natural. DM Serif Display headings + Source Sans body.
Dark + Glass Modern, techy, immersive. Backdrop blur, subtle gradients, floating elements.
Minimal + Light Clean, editorial, content-first. Generous whitespace, restrained color.

Key Phrases That Get Better Results

  • "Use as many pictures and visuals as possible" — Claude will add images, icons, and illustrations instead of walls of text
  • "Include mermaid charts for the [timeline/flowchart/org chart]" — Claude will generate Mermaid.js diagrams that render beautifully
  • "Make it a single HTML file with inline CSS and JS" — no build step, just open in a browser
  • "Use Google Fonts via CDN" — specify the fonts you want (Inter, Playfair Display, DM Serif Display, Source Sans 3)
  • "Add smooth scrolling and scroll-driven animations" — makes everything feel polished and alive
  • "Mobile responsive" — Claude adds media queries automatically
  • "Glass-morphism with backdrop-filter blur" — that frosted glass look for cards and overlays

Example Prompts from Real Projects

"I need a website that explains our portfolio company strategy to limited partners. Make it look like a premium strategy deck — dark background, gold accents, serif headings. Use /frontend-design for the best quality. Include mermaid charts showing our investment thesis flow. Single HTML file."

"Build me an internal dashboard for tracking our portfolio companies. I need a sidebar with navigation, a main area with summary cards showing total invested, current value, and MOIC, then a table of all companies below. Use our standard stack — Next.js, Supabase, Tailwind. Make it look executive, not startup-y."

"Create a trivia night event page for Young Leaders. Fun, vibrant, but still professional. Include the schedule, team registration info, and a merch section. Use images and keep the energy high. Deploy to GitHub Pages."

The Review Loop

After Claude builds something, always review:

  1. Open it in your browser to check visually
  2. Run /review-pr to have the PR Review Toolkit agents check code quality, test coverage, error handling, and type design
  3. If the review passes — ship it. Tell Claude to commit, push, and deploy.
  4. If the review flags issues — just say "fix the issues from the review" and Claude will address them
  5. Run /simplify to clean up the code after everything looks right

Iterate with Voice

Once you've got something on screen, use voice (spacebar) to quickly iterate: "make the hero bigger", "swap to a dark background", "add a section about pricing". Claude edits the file in place, so just refresh your browser to see changes.

Single-file HTML is king. For explainers, landing pages, and internal tools — one self-contained HTML file with inline CSS/JS is the fastest path. No build step. No dependencies. Just open in a browser. Save the full-stack (Next.js + Supabase) for projects that actually need a database and auth.
